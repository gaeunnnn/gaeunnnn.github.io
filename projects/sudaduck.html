TROUBLE SHOOTING

트러블 슈팅

가장 오래 고민한 문제와 해결 과정을 중심으로 정리했습니다.

#01

동시 음성 입력 시 대화 순서 보장

문제

동시 발화 상황에서 메시지 저장 순서가 뒤섞여 실제 대화 흐름이 깨졌습니다.

원인

병렬 요청 처리 과정에서 공통된 순서 기준이 없어 도착 순서와 발화 순서가 달랐습니다.

해결

Redis INCR로 방 단위 sequence를 발급하고, STT 처리·저장·스크립트 생성 모두 이 번호 기준으로 정렬하도록 통일했습니다.

결과

동시 발화에서도 순서 일관성을 유지했고 리뷰/복습 데이터 품질이 안정화되었습니다.

#02

STT 오인식으로 인한 스크립트 품질 저하

문제

STT 오인식 문장이 그대로 반영되어 학습 스크립트 자연도가 떨어졌습니다.

원인

음성 인식 결과를 검증 없이 LLM 단계로 전달해 노이즈가 그대로 확장됐습니다.

해결

중복/잡음/비정상 구문을 제거하는 규칙 기반 전처리를 추가하고, 정제 완료 텍스트만 생성 파이프라인에 투입했습니다.

결과

스크립트 문장 정확도와 자연도가 개선되어 복습 콘텐츠 신뢰성이 높아졌습니다.

#03

실시간 상태와 영속 데이터의 책임 분리

문제

실시간 상태와 최종 학습 데이터를 같은 경로로 처리해 응답성과 구조가 함께 저하됐습니다.

원인

즉시성 중심 데이터와 보존 중심 데이터의 저장 책임이 분리되어 있지 않았습니다.

해결

방 상태/참여자 상태는 Redis에서 즉시 처리하고, 학습 결과/스크립트는 DB에 영속화하는 이중 저장 전략으로 분리했습니다.

결과

상태 조회 지연이 줄고 저장 구조가 명확해져 유지보수성과 확장성이 개선되었습니다.
