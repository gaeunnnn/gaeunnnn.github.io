PROJECT DETAIL

Web Project 2025.11 ~ 2025.12

EnjoyTrip - 다시, 여행

여행의 준비부터 기록까지 연결하는 협업 여행 플랫폼

여행 정보를 탐색하고, 일정을 함께 구성하며, 여행 후에는 AI로 여행 기록까지 남길 수 있도록 기획된 서비스입니다. 협업형 일정 관리와 기록 흐름을 고려한 구조 설계에 집중했습니다.

GitHub 프로젝트 목록

프로젝트 개요

관광지 탐색, 일정 드래그 앤 드롭 편집, 실시간 협업, 여행 후 AI 여행일기 생성까지 여행 전 과정의 흐름을 하나로 연결한 서비스입니다. 여러 사용자가 함께 일정을 구성하는 협업 기능이 핵심이었습니다.

기술 스택

Java Spring Boot JPA MySQL Redis React AI Diary Workflow

내 역할

  • 여행 계획 및 일정 관리 기능을 위한 API 설계
  • 협업형 일정 데이터 구조 및 흐름 설계
  • 여행 후 기록 기능을 고려한 서비스 구조 설계
  • 프로젝트 데이터 모델링 및 백엔드 개발 참여

중점적으로 고민한 부분

  • 일정 데이터가 사용자 단독이 아니라 협업 단위로 움직인다는 점
  • 여행 준비 단계와 기록 단계가 하나의 서비스 흐름으로 이어져야 한다는 점
  • 관광지, 일정, 사용자, 여행 기록 데이터를 어떻게 연결할지

트러블 슈팅 / 설계 관점

1. 협업형 일정 데이터 구조

일정은 개인 데이터가 아니라 여러 사용자가 함께 수정하는 협업 데이터이기 때문에 누가 어떤 여행 일정에 참여하고 있는지, 어떤 순서로 장소가 배치되는지를 구조적으로 표현하는 것이 중요했습니다. 이를 위해 일정 단위, 참여 단위, 장소 배치 단위를 구분해 생각하며 데이터 구조를 설계했습니다.

2. 여행 후 기록까지 이어지는 서비스 흐름

이 서비스는 여행 계획으로 끝나지 않고 여행 후 AI 여행일기 생성까지 이어지는 것이 특징이었습니다. 그래서 단기적인 일정 저장이 아니라, 이후 기록과 회고 기능으로 연결될 수 있는 데이터 흐름을 고려해야 했습니다.

배운 점

협업형 서비스에서는 데이터 하나를 저장하는 것보다 사용자 간 관계와 흐름을 어떻게 모델링하는지가 더 중요하다는 점을 배웠습니다. 또한 서비스의 현재 기능뿐 아니라 이후 확장될 기능까지 고려해 구조를 설계하는 경험을 할 수 있었습니다.
