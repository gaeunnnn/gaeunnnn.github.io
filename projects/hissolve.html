TROUBLE SHOOTING

트러블 슈팅

핵심 문제를 문제-원인-해결-결과 흐름으로 정리했습니다.

#01

Connection Pool 계산 오류와 deadlock

문제

pool size를 크게 잡았음에도 특정 부하에서 deadlock이 발생했습니다.

원인

요청당 connection 사용량을 단순 계산해 트랜잭션 점유 패턴을 놓쳤습니다.

해결

max pool size를 낮춘 극단 테스트로 병목 구간을 식별하고, 트랜잭션/쿼리 흐름을 재점검했습니다.

결과

자원 사용 기준이 명확해졌고 운영 환경에서 pool 설정 근거를 확보했습니다.

#02

동시 제출 시 임시 파일 충돌

문제

동시 제출 상황에서 서로의 코드/실행 결과가 섞이는 현상이 발생했습니다.

원인

모든 제출이 동일한 임시 디렉토리를 사용해 파일 덮어쓰기가 발생했습니다.

해결

제출마다 UUID 기반 고유 경로를 생성해 컴파일/실행 경로를 완전히 분리했습니다.

결과

동시 제출에서도 충돌 없는 독립 실행이 가능해져 채점 신뢰성이 개선됐습니다.

RETROSPECTIVE

배운 점

Hissolve를 통해 대회 플랫폼은 기능 구현보다 동시성 상황에서의 안정성이 더 중요하다는 점을 배웠습니다. 특히 실행 환경 격리, connection pool 검증, 병목 분석 같은 운영 관점의 문제 해결이 백엔드 신뢰성을 좌우한다는 것을 실전에서 체감했습니다.
